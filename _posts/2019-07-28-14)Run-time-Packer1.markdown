# 14.1) 데이터 압축  
## 14.1.1) 비손실 압축  
* 압축 해제시 원래 파일(데이터)을 얻어낼 수 있다.
* 파일(데이터) 크기를 줄여서 보관 및 이동에 용이하도록 하려는 목적  
* Run-length, Lempel-Ziv, Huffman 등이 있다.  

## 14.1.2) 손실 압축  
* 압축 해제시 원래 파일(데이터)을 얻어낼 수 없다.  
* 데이터에 의도적인 손상을 주어 압축률을 높이는 것이 목적  
* 주로 멀티미디어 파일에 많이 씀.  
* mp3, jpg, mp4 등이 있다.  

# 14.2) 실행 압축  
* 실행 압축: 실행(PE) 파일을 대상으로 파일 내부에 압축해제 코드를 포함하고 있어서 실행되는 순간에 메모리에서 압축을 해제시킨 후 실행시키는 기술이다.  
* 내부는 원본 PE 파일 + decoding 루틴으로 이루어져있다.  
* 별도의 해제 프로그램 없이 바로 실행 가능하다.  
* 일반 압축과 가장 두드러지는 차이점은 **PE파일의 실행이 가능하다**는 것이다.(.zip 파일같은건 그대로 실행못한다는걸 생각하면 편하다.)  
* 일반 PE 파일을 실행 압축 파일로 만들어 주는 유틸리티를 *패커*라고 한다.
* Anti-Reversing 기법에 특화된 패커를 *프로텍터*라고 한다.  

## 14.2.1) 패커  
1. 사용 목적  
* PE 파일의 크기를 줄이고자 하는 목적  
* PE 파일의 내부 코드와 리소스를 감추기 위한 목적  

2. 패커 종류  

|순수한 의도| 불순한 의도|  
|:--------|:-----------|  
|UPX, ASPack|UPack, PESpin, NSAnti|  

## 14.2.2) 프로텍터  
1. 사용 목적  
* 크래킹 방지  
* 코드 및 리소스 보호(덤프조차 뜨지 못하게 만든다.)  

2. 프로텍터 종류  

|상용 프로텍터 | 공개 프로텍터 |  
|:-----------|:--------------|  
|ASProtect, Themida, SVKP| UltraProtect, Morphine |
